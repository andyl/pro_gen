# ProGen Design Notes - Version 1

ProGen - A scriptable Elixir Project Generator

ProGen embraces [Igniter](), for elixir code generation, and also manages tasks
beyond the scope of Igniter, like Provisioning, Deployment, CICD, Monitoring & DevOps.

Key Elements

- Operations | Composable elements that perform atomic generation tasks
- Scripts    | Sharable/composable scripts for end-to-end project generation
- Menus      | User-selectable choices to create personalized ProGen script
- Chats      | LLM chat to create personalized ProGen script  

## Argument Validation

We need to validate arguments for two use cases:
- Scripts    | Need to validate CLI arguments and generate usage/help messages
- Operations | Need to validate function call arguments.  Ideally this would work
  in the editor via ExpertLSP.

### Decision: NimbleOptions (Operations) + Optimus (Scripts)

Optimus is tightly coupled to CLI string argv parsing and can't directly validate
keyword lists/maps. Therefore we use two complementary tools:

- **NimbleOptions** — Used for Operation argument validation. Each operation defines an
  `option_schema/0` callback returning a NimbleOptions schema. Validation and
  auto-generated usage docs come for free. Already a transitive dep via
  igniter → finch, now listed as a direct dep.
- **Optimus** — Reserved for future Script CLI argv parsing with auto-generated
  help/usage text. Added as a dep but not yet wired into `ProGen.Script`.

## Operations

Overview

- each operation is a module (ProGen.Operation.<Name>) that invokes `use ProGen.Operation`.
- operations can compose other operations
- operations are self-descriptive (like mix tasks and LV components)
- operation behavior is defined in ProGen.Operation (perform, description, usage)

BuiltIn Operations
- args  | Arg parser with Usage Message
- mix   | run mix task
- run   | run a system command
- file  | create a file
- ask   | get feedback
- yes?  | return t/f
- no?   | return t/f
- route | add a route

## Scripts

Overview

- plain elixir scripts — no module definition required
- `Mix.install([{:pro_gen, "github:andyl/pro_gen"}])`
- call functions as `ProGen.Script.msg(...)` or `import ProGen.Script` 
- can be written by end-users
- can be generated by menus (FUTURE)

### Script DSL

`ProGen.Script` provides plain functions (no macros, no callbacks):

- `parse_args(schema, argv)` — Parse argv (list or string) against an Optimus schema. Returns `{:ok, parsed}` / `{:error, errors}` / `:help` / `:version`.
- `usage(schema)` — Generate help text from an Optimus schema.
- `msg(text)` — Print a formatted message.
- `cmd(desc, command)` — Print description then run a system command.
- `op(desc, operation, opts)` — Run a ProGen operation (stub).
- `git(arg)` — Run a git command (string or list).
- `commit(message)` — Stage all and commit.

### System Commands

`ProGen.Sys` provides low-level system command wrappers:

- `syscmd(cmd)` — Parse and run a command string.
- `syscmd(cmd, args)` — Run a command with an explicit arg list.
